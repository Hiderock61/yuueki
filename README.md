# YUUEKi.com

👵🏻 **つながろおばちゃんのガイド付き**

YUUEKi.com は、人間関係の目的を最初から正直に置ける、一対一ルーム型の静的UIプロトタイプです。

恋愛のふりをしなくていい。
ただ話したい、会いたい、相談したい、距離を取りたい。
そういう目的を、境界線つきで最初から置ける場所として作っています。

---

## 現在地

このリポジトリは、**YUUEKi.com の静的Webアプリ試作版**です。

現時点では、実運用サービスではありません。
ログイン、会員登録、本人確認、保存機能、外部API、実際のマッチング機能は前提にしていません。

まずは、以下の体験を確認するためのUI模型です。

* 目的を最初に言えるか
* 断る自由を画面内に置けるか
* おばちゃんが間に入ると会話の圧が下がるか
* トリセツを作ることで、自分のOK/NGを整理できるか
* AIが本人のふりをせず、補助役として座れるか

---

## コンセプト

### 目的を言う自由。断る自由。

YUUEKi.com の中心は、次の2つです。

```text
自分の目的を言う自由
相手が断る自由
```

この2つをセットで置きます。

片方だけだと、圧が強くなります。
目的を言うだけだと押しつけになる。
断る自由だけだと、最初から何も始まらない。

だから YUUEKi.com では、目的と撤退権を同じ画面に置きます。

---

## つながろおばちゃんとは

つながろおばちゃんは、会話の相手のふりをするAIではありません。

おばちゃんの役割は、あくまで間に入ることです。

* 最初の説明を助ける
* 気まずさを少し下げる
* 境界線を見えるようにする
* 会話が止まった時に助け舟を出す
* 断りやすくする
* 目的がズレた時に整理する

本人の代わりに恋愛したり、本人のふりをして会話したりはしません。

---

## このアプリでできること

現時点の試作では、以下のような流れを確認できます。

### 1. トリセツを作る

自分のOK/NG、話しやすいこと、苦手なこと、距離感などを整理します。

プロフィールではなく、**扱い方のメモ**です。

### 2. おばちゃん介入度を選ぶ

おばちゃんにどれくらい間に入ってほしいかを選びます。

例：

* そっと見守る
* 困った時だけ助ける
* ほどよく間に入る
* かなり世話を焼く

### 3. 仮想ルームを試す

一対一ルームの雰囲気を確認します。

これは本物のチャットサービスではなく、会話体験を試すためのデモです。

### 4. 今日は見るだけ

登録や入力を強制せず、見るだけで終われる入口を置いています。

## 大事な方針

### AIは本人の代筆をしない

YUUEKi.com では、AIが本人のふりをして相手に話しかける設計にはしません。

AIは、次のような補助に使います。

* 自分の言葉を整理する
* 言い方をやわらかくする
* 断り文を作る
* 会話の目的を整理する
* 境界線を見えるようにする

本人の意思や同意を飛ばして、AIが勝手に進める設計にはしません。

### 合意と撤退権を前提にする

YUUEKi.com は、成人同士の合意と撤退権を前提にしたUI実験です。

* 目的を言ってよい
* 断ってよい
* 途中でやめてよい
* 返事を急がなくてよい
* 会わない選択も残す

このあたりを、画面構造の中に入れることを重視しています。
