# PROJECT_CARD

## このアプリは何？
YUUEKi.com はブラウザだけで動くクライアントサイドの対話プロトタイプです。利用者が「トリセツ（自己のOK/NGや目的）」を作成して待合室で目的や時間を選び、仮想相手と一対一の「茶の間」ルームで会話をするUXを提供します。おばちゃん（助け舟）による介入や、会話後のふりかえり機能を備えたプロトタイプ（app.js に Ver.0.6-A の表記あり）。

根拠ファイル：index.html / app.js / style.css

## 現在の状態
- 実行形のフロントエンド（単一ページ）として画面群が実装済み：
  - トップ、トリセツ作成（6問）、介入度選択、待合室（目的・時間選択）、相手選択、仮想一対一ルーム、助け舟（おばちゃん）メニュー、あみだ（ランダム選択）、ふりかえりフロー 等。
- UI と挙動は HTML + JS + CSS で完結しており、ローカルメモリ（userManual / state）に状態を保持する設計。永続化やサーバー連携の痕跡はない。
- ドキュメント（README、LICENSE 等）はリポジトリに存在しない（トップレベルは index.html / app.js / style.css のみ）。
- バージョン表記や TODO コメントはコード内にあり、プロトタイプ段階であることが示される。

## 状態ラベル
- 実験（プロトタイプ）：機能は広く作り込まれているが、運用・公開ポリシーや永続化・モデレーションが未整備のため試験的。

## 主な機能
- トリセツ（ユーザーのOK/NG・目的・会話温度）作成（6問）と確認画面
- 待合室フロー：目的選択、話せる時間選択
- 仮想相手プリセット（gentle / energetic / serious / casual 等）
- 仮想一対一ルーム（チャット表示、メッセージ生成、相手のモック返信）
- おばちゃん（助け舟）機能：呼び出しアクション（待つ、話題変更、介入、終了）
- あみだ（ランダム選択）と整理棚（高リスク時の誘導）
- 会話後のふりかえりとメモの一時保存（メモリ内、userManual.learning）
- 多数の UI コンポーネントと状態管理ロジック（app.js）

## GitHub Pages公開
- 公開可能：静的ファイル（index.html, app.js, style.css）だけで動作するため、そのまま GitHub Pages にデプロイ可能。ビルド不要。Pages の公開ブランチ設定（main 等）を確認するだけでデモURLが作れる見込み。

## 足りないもの

- プライバシー / セーフティ注意（成人向けの選択肢が存在するため注意喚起・年齢制限等）
- 永続化・バックエンドやデータ保護に関する設計（現状はメモリ内のみ）
- テスト、CI、アクセシビリティ確認ドキュメント
- 運用ルール（モデレーション、緊急時対応、個人情報の取り扱い）
- スクリーンショットや短いデモ手順（README 用）
